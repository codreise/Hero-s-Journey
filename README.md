# Hero’s Journey

Hero’s Journey — це хвильова pixel RPG на React + Vite з підтримкою Telegram Mini App, локальних і серверних збережень, босів, нагород між хвилями та мета-прогресу між забігами.

## Що є в проєкті

- фронтенд на React і Vite
- canvas-бойовка з мобільним і клавіатурним керуванням
- локальний save-сервер на Node.js
- підтримка Telegram Mini App через ngrok для локального тестування
- мета-прогрес, боси, нагороди між хвилями, пауза і продовження забігу

## Швидкий старт

1. Встановити залежності:

```powershell
npm install
```

2. Запустити фронтенд:

```powershell
npm run dev
```

3. Відкрити гру в браузері:

```text
http://localhost:5173
```

4. Якщо потрібні серверні збереження для Telegram, у другому терміналі запустити:

```powershell
npm run server
```

## Telegram-бот

У репозиторії є мінімальний бот у [bot/index.js](bot/index.js), який після `/start` або `/play` надсилає повідомлення з inline-кнопкою Грати.

Запуск:

```powershell
$env:BOT_TOKEN="your_bot_token"
$env:WEB_APP_URL="https://your-mini-app-url"
npm run bot
```

## Основні механіки

- Хвильовий бій: кожна хвиля завершується після знищення всіх ворогів.
- Нагорода між хвилями: після бою гравець обирає один із трьох бонусів.
- Бос-хвилі: кожна п’ята хвиля запускає окремий бій з босом.
- Прокачка під час забігу: атака, здоров’я, зілля, монети, XP.
- Мета-прогрес: після завершення забігу гравець отримує есенцію й купує постійні покращення.
- Серверні збереження: у Telegram прогрес може зберігатися за user id.

## Типові проблеми

Якщо не працюють Telegram-збереження:

1. Переконайся, що запущено `npm run server`.
2. Переконайся, що фронтенд запущений до старту ngrok.
3. Перевір [vite.config.js](vite.config.js) і список `allowedHosts`.
4. Відкрий `http://localhost:3001/game-api/health` і перевір, що сервер відповідає `{ "ok": true }`.

Якщо бачиш білий екран:

1. Подивись помилки в терміналі з `npm run dev`.
2. Запусти `npm run build`.
3. Перевір діагностику в папці [src/components/game](src/components/game).
